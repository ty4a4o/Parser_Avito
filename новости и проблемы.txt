Функции
main:
    init_db(): Инициализация базы данных. Создает таблицы users (пользователи), orders (заказы) и task_logs (история выполнений), если они отсутствуют.
    get_db_connection(): Служебная функция для создания подключения к файлу backend.db с поддержкой имен колонок (Row).
    register(UserRegister): Регистрация нового пользователя. Хеширует пароль через bcrypt и записывает данные в БД.
    login(UserLogin): Проверка учетных данных. Сравнивает введенный пароль с хешем в базе.
    create_order(OrderCreate): Создание заказа. Проверяет баланс пользователя, списывает баллы и добавляет запись в таблицу заказов.
    get_user(user_id): Получение актуальных данных о пользователе (включая текущий баланс) по его ID.
    get_all_orders(): Возвращает список всех активных заказов в системе для мониторинга.
    get_available_tasks(user_id): Фильтрует заказы, которые пользователь еще не выполнял за последние 24 часа.
    complete_task(TaskComplete): Фиксация выполнения задания. Начисляет +5 баллов исполнителю, прибавляет +1 к счетчику заказа и ставит «кулдаун» в лог.
gui:
    __init__(root): Настройка главного окна, инициализация переменных и запуск окна авторизации.
    show_login_window() / show_register_window(): Создание всплывающих окон для входа и регистрации с полями ввода.
    process_login(): Собирает данные из полей, отправляет запрос на сервер и переключает интерфейс на «Главное меню».
    setup_main_ui(): Отрисовка основного интерфейса с вкладками (Заказчик/Исполнитель), верхней панелью баланса и кнопкой выхода.
    start_creation_with_script(): Запускает «многопоточность» (threading), чтобы интерфейс не «зависал» на время работы браузера.
    process_order_creation(): Главная логика кнопки:
        1.Вызывает проверку ссылки через act.py.
        2.При успехе отправляет запрос в API на создание заказа.
        3.Очищает поля и обновляет баланс на экране.
    update_user_info(): Обновляет локальные данные пользователя и текст баланса (Label) в верхнем углу.
    load_all_orders() / load_available_tasks(): Загружает данные из API и заполняет таблицы (Treeview) актуальными заказами.
    complete_task_logic(): Логика кнопки «ВЫПОЛНИТЬ». Отправляет ID задания на сервер и обновляет данные при успехе.
act:
    setup_driver(): Настройка undetected_chromedriver. Подменяет User-Agent и параметры окна, чтобы сайт не распознал бота.
    run_validation(target_url): Главная точка входа для GUI. Принимает ссылку, открывает её и запускает сценарий действий. Возвращает True/False.
    micro_pause(): Создает случайные задержки между действиями (от 1.1 до 2.5 сек) для имитации естественного поведения.
    check_browser(driver): Проверяет, не закрыл ли пользователь окно браузера вручную во время работы скрипта.
    Сценарии взаимодействия:
        1.Прокрутка страницы: Эмуляция чтения текста.
        2.Переключение фото: Клик по галерее изображений.
        3.Поиск: Ввод текста в поисковую строку Avito (если ссылка ведет на общую страницу).
